# l-picola-nongaussian-pdf

Modified implementation of L-PICOLA for cosmological simulations with PDF-based primordial non-Gaussian initial conditions using external random-field input.

---

## Overview

This repository contains a modified version of the L-PICOLA code designed to generate cosmological initial conditions for a class of primordial non-Gaussian models defined at the level of the full probability distribution function (PDF).

The main modification consists of replacing the standard Gaussian random mode generation in Fourier space with an externally generated random field. This enables the construction of initial conditions beyond standard bispectrum-based parametrizations (e.g. f_NL models).

---

## Methodology

In the original L-PICOLA implementation, Gaussian initial conditions are generated by sampling Fourier modes using random amplitudes and phases:

$\delta(k) \sim \sqrt(P(k)) * \sqrt(-\log U) * \exp(i \phi)$

where U is a uniform random variable and phi is a random phase.

In this modified version:

- Random numbers are generated independently from the simulation code
- The random field follows an arbitrary target probability distribution function (PDF)
- The field is read from an external binary file
- A 3D FFT is applied to obtain Fourier modes
- These modes are used to construct the displacement field

Conceptually, this replaces:

delta(k) ~ internally generated Gaussian random field

with:

delta(k) ~ FFT[input random field]

This approach allows the generation of non-Gaussian initial conditions defined directly at the PDF level, rather than through low-order correlators such as the bispectrum.

## Main Modifications

### 1. External random field input
- Implemented in: `read_random_number_file.h`
- Reads random numbers from a binary file
- Input field can be generated using arbitrary sampling methods (e.g. acceptance–rejection)

---

### 2. Modified 2LPT pipeline
- File: `2LPT.c`
- Replaces Gaussian amplitude–phase sampling with FFT of input random field
- Uses Fourier coefficients from the input field instead of sin/cos phase construction
- Enables PDF-based primordial non-Gaussian initial conditions

## Configuration

The type of initial conditions and the source of the random field are controlled at compile time through the Makefile using two parameters:

- **IC_TYPE**: defines the statistical model of the initial conditions  
- **RAND_SOURCE**: defines how the random field is generated  

### IC_TYPE options

- `GAUSSIAN` — standard Gaussian initial conditions  
- `LOCAL_FNL` — local-type primordial non-Gaussianity  
- `EQUIL_FNL` — equilateral-type non-Gaussianity  
- `ORTHO_FNL` — orthogonal-type non-Gaussianity  
- `GENERIC_FNL` — generic kernel-based implementation  

### RAND_SOURCE options

- `INTERNAL` — standard internal Gaussian random number generation  
- `EXTERNAL` — random field is read from an input file  

The external random field option enables the generation of initial conditions from arbitrary probability distribution functions (PDFs).

**Note:** the external random field implementation is currently supported for the Gaussian branch.

## Usage

### 1. Compile

Ensure the flag `-DRANDOM_NUMBER_FILE` is enabled in the Makefile, then compile as usual.

---

### 2. Provide random field

- Input must be a binary file containing the random field
- The field should:
  - follow the desired PDF
  - be properly normalized
  - match the simulation grid size

---

### 3. Run

Run L-PICOLA as usual using the modified parameter file.

---

## Random Field Generation

This repository does not include a built-in random number generator. Instead, it expects an externally generated random field as input.

This design allows full flexibility in how the field is generated. Any sampling method can be used, including:

- acceptance–rejection sampling
- inverse transform sampling
- custom numerical methods

A dedicated random-field generator may be provided in future work.

---

## Important Notes

- Proper normalization of the FFT is required for correct amplitudes
- The statistical properties of the simulation depend entirely on the input random field
- Parallel performance of the input routine has not been fully tested

## Scientific Context

This implementation enables the study of cosmological structure formation with non-Gaussian initial conditions defined at the level of the full PDF, motivated by inflationary scenarios where deviations from Gaussianity cannot be fully captured by low-order correlators.
